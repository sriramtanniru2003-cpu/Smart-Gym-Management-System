# Gym Management System

A comprehensive full-stack application for managing gym operations, including member registration, attendance tracking, trainer management, and class scheduling.

## Project Overview

The Gym Management System is a modern web application designed to streamline gym operations and improve member experience. It provides separate dashboards for admins, trainers, and members with role-based access control.

## Project Structure

## Technology Stack

### Backend
- **Runtime**: Node.js
- **Framework**: Express.js
- **Database**: MongoDB
- **Authentication**: JWT (JSON Web Tokens)
- **Password Hashing**: bcrypt

### Frontend
- **Framework**: React 19
- **Build Tool**: Vite
- **Styling**: Tailwind CSS
- **HTTP Client**: Axios
- **Routing**: React Router v7
- **Charts**: Recharts
- **QR Code Scanner**: html5-qrcode
- **Icons**: Lucide React, React Icons

## Getting Started

### Prerequisites
- Node.js (v14 or higher)
- MongoDB instance
- npm or yarn package manager

### Environment Variables

Create a `.env` file in the backend root directory:

\`\`\`env
# Backend Configuration
MONGO_URI=mongodb://localhost:27017/gym_management
PORT=3000
NODE_ENV=development
JWT_SECRET=your_secret_key_here
\`\`\`

Create a `.env` file in the frontend root directory:

\`\`\`env
# Frontend Configuration
VITE_API_URL=http://localhost:3000/api
\`\`\`

### Installation & Running

#### Backend

\`\`\`bash
cd gym_backend
npm install
npm run dev      # Development mode with hot reload
npm start        # Production mode
\`\`\`

The backend will run on `http://localhost:3000`

#### Frontend

\`\`\`bash
cd gym_frontend
npm install
npm run dev      # Development server
npm run build    # Build for production
npm run preview  # Preview production build
\`\`\`

The frontend will run on `http://localhost:5173`

## API Endpoints

For detailed API documentation, see [Backend README](./gym_backend/README.md).

### Authentication
- `POST /api/auth/signup` - Register new user
- `POST /api/auth/login` - User login

### Members
- `GET/POST /api/member` - Get/create members
- `GET /api/member/:id` - Get member details

### Trainers
- `GET/POST /api/trainer` - Get/create trainers
- `GET /api/trainer/:id` - Get trainer details

### Classes
- `GET/POST /api/classes` - Get/create classes
- `GET /api/classes/:id` - Get class details

### Attendance
- `POST /api/attendance/generate-qr` - Generate QR code
- `POST /api/attendance/mark-qr` - Mark attendance via QR

## Features in Detail

### QR Code Attendance
Members and trainers can mark attendance using QR codes generated by the system. This streamlines the check-in process and reduces manual errors.

### Role-Based Access Control
The application implements role-based authorization with three main roles:
- **Admin**: Full system access
- **Trainer**: Manage assigned members and classes
- **Member**: Access personal dashboard and attendance

### JWT Authentication
Secure token-based authentication ensures protected routes and API endpoints.

## Development

### Project Structure Conventions
- Controllers handle business logic and request validation
- Models define MongoDB schemas and relationships
- Routes define API endpoints
- Middlewares handle authentication, authorization, and validation

## Troubleshooting

### Common Issues

1. **MongoDB Connection Error**
   - Ensure MongoDB is running
   - Verify `MONGO_URI` in `.env` is correct

2. **CORS Errors**
   - Check frontend API URL configuration
   - Verify backend CORS settings

3. **Authentication Issues**
   - Ensure JWT_SECRET is set in backend `.env`
   - Check token expiration settings
